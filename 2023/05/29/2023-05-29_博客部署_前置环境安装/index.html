博客背景介绍

​ 本博客是由hexo生成的纯静态页面的html代码，使用butterfly模板作为基础来进行改进而来的。他不需要服务器的支撑，会减少购买服务器等方面的费用，这也是我选择它的原因。源代码完全由github进行管理，只需要你访问仓库地址即相当于访问你的博客。再配置过程中，往往需要下载一些组件以及提交代码到github，所以需要安装node、git等开发工具，下面详细讲述。


node安装

  1. node可以去他的官网下载 node.js，选择对应的版本后，点进去下载相关的系统安装程序，windows下载msi后缀的程序，X86是32位，X64是64位。具体如下图


  1. 下载完node安装包之后便进行安装，这里选择的是你的node安装位置，默认是c盘，你可以根据自己的想法进行修改；然后一直点击下一步直至安装完成。


  1. 然后你在电脑上找到运行或者win+r打开命令框输入cmd命令，打开命令窗口。输入 node -v命令可以看到有显示版本信息，输入npm -v也会有输出对应的版本信息。那就证明你已经安装完成可以使用了。

  1. 现在你们全局的仓库是安装在c盘下的，可以通过npm list -global命令查看；所以当你们使用npm安装全局模块的时候会安装到c盘下面的文件夹，这会使得你们c盘数据庞大，所以我们便可以修改默认存储地址；但是如果你们c盘本身比较大，不在乎多存的这点数据，那就可以忽略不计这一步。在你安装node的文件夹下新建两个文件夹；node_cache(缓存文件夹)node_global(全局仓库文件夹)

​ 然后再命令行输入

​ 这时候你的全局仓库地址和缓存地址就已经修改完成；你可以使用命令行安装某个东西测试一下；比如我全局安装一下webpack，你就会看到node_global下面出现了webpack的文件夹，以后你全局安装的任何模块都可以再这个文件夹里面找到。

  1. 添加环境变量，进入环境变量，编辑【系统变量】下的变量【Path】

​ 注意：由于 node 全局模块大多数都是可以通过命令行访问的，还要把【node_global】的路径“E:\devTools\nodejs\node_global”加入到【系统变量 】下的【PATH】 变量中，方便直接使用命令行运行，如下图所示：

  1. 设置淘宝镜像，将npm默认的registry修改为淘宝registry说明：npm 默认的 registry ,也就是下载 npm 包时会从国外的服务器下载，国内下载会很慢，可以更换为国内镜像： https://registry.npmmirror.com。

    b. 更换npm为国内镜像

    c. 检查镜像是否配置成功,再次执行npm config get registry，检查当前的镜像路径。如上图所示，npm默认的registry已修改为国内镜像registry。


git安装

​ 这个就需要去 Git 官网下载对应系统的软件了，下载地址为 git-scm.com或者gitforwindows.org，或者阿里镜像。这里的 git-scm 是 Git 的官方，里面有不同系统不同平台的安装包和源代码，而 gitforwindows.org 里只有 windows 系统的安装包。阿里镜像直接Ctrl+F 搜索最新版本前缀即可，如2.40.0
